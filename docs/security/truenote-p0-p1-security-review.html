Decision summary

Good control foundation, not a certification claim.P0 and P1 application controls are implemented in this worktree. Do not describe Truenote as “FedRAMP compliant,” “FedRAMP-ready,” or “fully controlled ingestion.” Database migration, configuration, operational evidence, vendor assurance, infrastructure scope, and independent assessment remain incomplete.

FedRAMP describes Class C as the 20x class for common enterprise or important government services. Its 2026 rules require persistent evidence and independent verification, including annual assessment of all Key Security Indicators for Class C. This packet uses Class C as a conservative internal design target, not as a FedRAMP determination.

Official sources: FedRAMP 20x, 2026 Class C independent verification rules, Authorization by FedRAMP KSIs, and 2026 identification and authentication guidance.

Implemented P0/P1 controls

01

Controlled lifecycle

Nothing activates automatically. Uploads remain inactive through scan, parse, and review.

02

Fail-closed scanning

File signatures, EICAR, malware, secrets, SSNs, payment cards, and prompt-injection patterns are checked.

03

DLP before providers

Blocked sources quarantine before embedding. Sensitive ask input and model output are blocked.

04

Human approval

Review includes provenance, parsed text, findings, and classification. Uploaders cannot self-approve.

05

Classification

Server-owned clearance filters every retrieval and knowledge-base read path.

06

OIDC and MFA evidence

PKCE, signed state, nonce, RS256, issuer, audience, authorized party, domain, MFA, and optional ACR checks.

07

Security audit

Hash-chained ledger, mutation trigger, lifecycle triggers, request correlation, and signed SIEM export.

08

Retention and revocation

Retire preserves evidence. Revoke removes retrieval and citations. Purge is retention-gated.

09

Build assurance

Typecheck, tests, audit, SBOM, Gitleaks, Dependabot, and a zero-finding CodeQL SARIF gate with 30-day evidence retention.

What remains lacking

P0 deployment blockers

  • Apply and verify DDL in non-production, then production through an approved change window.
  • Select an approved scanner and validate location, retention, subprocessors, incident terms, authentication, and uptime.
  • Configure IdP OIDC and phishing-resistant MFA. Verify amr: ["mfa"] or an approved ACR.
  • Configure signed SIEM delivery and alert on authentication, quarantine, activation, revocation, purge, limit, and exporter failures.
  • Review seeded sources, legacy documents, user clearances, and provisional 365-day retention.
  • Run typecheck, tests, integration, negative security, and staged smoke tests.

P1 organizational and infrastructure gaps

  • GitHub-integrated CodeQL alerts require an eligible organization plan with GitHub Code Security for this private repository. Until then, CI runs the security-extended suite, requires zero SARIF findings, and retains the SARIF evidence for 30 days.
  • Define Minimum Assessment Scope, service boundary, data flows, assets, identities, ports, and customer responsibilities.
  • Obtain assurance for Replit, Neon, storage, model routes, embeddings, Cohere, LandingAI, email, scanner, SIEM, and subprocessors.
  • Confirm data location, encryption, key ownership, backup, disaster recovery, vulnerability, incident, personnel, change, and SDLC controls.
  • Determine FIPS-validated cryptography requirements and collect module evidence.
  • Create persistent KSI measurements, evidence retention, quarterly review, incident communication, significant change, and authorization data sharing.
  • Engage a FedRAMP Recognized independent assessment service. Class C requires verification at least annually.
  • Create the Security Decision Record and certification package; own remediation findings.

Practical rollout

  1. Approve decisions. Security, IAM, Legal/Records, data governance, and platform owners confirm defaults.
  2. Prepare non-production. Snapshot DB, inventory users/documents, configure test scanner and SIEM, keep local login enabled.
  3. Apply DDL. Give Replit Agent the prompt and exact SQL below. Save verification results.
  4. Deploy non-production code. Confirm missing controls fail closed.
  5. Run negative tests. Wrong signatures, EICAR, scanner outage, secrets/PII, prompt injection, self-approval, cross-program, over-classification, rate, revoke, purge, SIEM.
  6. Enable SSO safely. Test ordinary and privileged users plus failure cases, then switch local login to break glass.
  7. Review migrated data. Replace generic owners, approve clearances, review legacy active versions.
  8. Enable CI gates. Require checks and retain evidence.
  9. Production change. Snapshot, DDL, verify, secrets, deploy, smoke test, SIEM confirmation, release.
  10. Build persistent evidence. Track KSIs, access reviews, incidents, changes, vulnerabilities, recovery, vendors, exceptions.
No automatic down migration.This migration backfills governance metadata and creates audit evidence. Dropping it after production writes can destroy evidence. Use snapshot restore only under an approved incident/change decision; prefer a forward fix.

-- Truenote P0/P1 security controls
-- Target: Replit-managed PostgreSQL (Neon)
-- Apply in a maintenance window BEFORE deploying the matching application code.
-- Idempotent for repeated Replit Agent execution.

BEGIN;

-- Replit/Neon prerequisite used by the audit hash chain. The existing project
-- already uses gen_random_uuid(); this check makes a missing pgcrypto digest
-- fail with a clear message before any schema change is applied.
DO $$
BEGIN
  IF to_regprocedure('digest(text,text)') IS NULL THEN
    RAISE EXCEPTION 'pgcrypto digest(text,text) is required before P0/P1 DDL';
  END IF;
END $$;

-- 1. Approved content-source registry.
CREATE TABLE IF NOT EXISTS content_sources (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  program_id uuid NOT NULL REFERENCES programs(id) ON DELETE RESTRICT,
  name text NOT NULL,
  origin_type text NOT NULL CHECK (
    origin_type IN ('manual_upload', 'sharepoint', 'confluence', 's3', 'other')
  ),
  base_uri text,
  owner_name text NOT NULL,
  is_active boolean NOT NULL DEFAULT true,
  created_by uuid REFERENCES users(id) ON DELETE SET NULL,
  created_at timestamptz NOT NULL DEFAULT now(),
  approved_by uuid REFERENCES users(id) ON DELETE SET NULL,
  approved_at timestamptz,
  approval_basis text,
  retired_at timestamptz,
  CONSTRAINT content_sources_approval_check CHECK (
    (is_active = false) OR (approved_at IS NOT NULL AND owner_name <> '')
  )
);

CREATE UNIQUE INDEX IF NOT EXISTS content_sources_program_name_uidx
  ON content_sources (program_id, lower(name));
CREATE INDEX IF NOT EXISTS content_sources_program_active_idx
  ON content_sources (program_id, is_active, approved_at);

-- Bootstrap one controlled manual source plus one explicit legacy source per
-- existing program. Security/Data Steward must review owner and approval basis.
INSERT INTO content_sources (
  program_id, name, origin_type, owner_name, approved_at, approval_basis
)
SELECT
  p.id,
  'Manual administrator upload',
  'manual_upload',
  'Program data steward',
  now(),
  'P0/P1 bootstrap default; owner and authority require Security review'
FROM programs p
ON CONFLICT DO NOTHING;

INSERT INTO content_sources (
  program_id, name, origin_type, owner_name, approved_at, approval_basis
)
SELECT
  p.id,
  'Legacy migration source',
  'other',
  'Legacy owner review required',
  now(),
  'Existing content migrated into controlled lifecycle; formal provenance review required'
FROM programs p
ON CONFLICT DO NOTHING;

-- 2. Document and version lifecycle, provenance, classification, review,
-- scanner evidence, revocation, and retention.
ALTER TABLE documents
  ADD COLUMN IF NOT EXISTS lifecycle_state text NOT NULL DEFAULT 'active',
  ADD COLUMN IF NOT EXISTS retired_at timestamptz,
  ADD COLUMN IF NOT EXISTS retired_by uuid REFERENCES users(id) ON DELETE SET NULL,
  ADD COLUMN IF NOT EXISTS retirement_reason text;

ALTER TABLE document_versions
  ADD COLUMN IF NOT EXISTS lifecycle_state text NOT NULL DEFAULT 'submitted',
  ADD COLUMN IF NOT EXISTS classification text NOT NULL DEFAULT 'internal',
  ADD COLUMN IF NOT EXISTS source_id uuid REFERENCES content_sources(id) ON DELETE RESTRICT,
  ADD COLUMN IF NOT EXISTS source_origin_uri text,
  ADD COLUMN IF NOT EXISTS source_owner text,
  ADD COLUMN IF NOT EXISTS original_file_name text,
  ADD COLUMN IF NOT EXISTS scan_status text NOT NULL DEFAULT 'pending',
  ADD COLUMN IF NOT EXISTS scan_engine text,
  ADD COLUMN IF NOT EXISTS scan_id text,
  ADD COLUMN IF NOT EXISTS scan_findings jsonb NOT NULL DEFAULT '[]'::jsonb,
  ADD COLUMN IF NOT EXISTS scan_completed_at timestamptz,
  ADD COLUMN IF NOT EXISTS approved_by uuid REFERENCES users(id) ON DELETE SET NULL,
  ADD COLUMN IF NOT EXISTS approved_at timestamptz,
  ADD COLUMN IF NOT EXISTS approval_notes text,
  ADD COLUMN IF NOT EXISTS activated_at timestamptz,
  ADD COLUMN IF NOT EXISTS retired_at timestamptz,
  ADD COLUMN IF NOT EXISTS rejected_by uuid REFERENCES users(id) ON DELETE SET NULL,
  ADD COLUMN IF NOT EXISTS rejected_at timestamptz,
  ADD COLUMN IF NOT EXISTS rejection_reason text,
  ADD COLUMN IF NOT EXISTS revoked_by uuid REFERENCES users(id) ON DELETE SET NULL,
  ADD COLUMN IF NOT EXISTS revoked_at timestamptz,
  ADD COLUMN IF NOT EXISTS revocation_reason text,
  ADD COLUMN IF NOT EXISTS retention_until timestamptz NOT NULL DEFAULT (now() + interval '365 days');

-- Backfill every pre-control version. `legacy_accepted` is deliberately
-- distinguishable from a real clean scanner verdict.
UPDATE document_versions dv
SET
  source_id = COALESCE(
    dv.source_id,
    (
      SELECT cs.id
      FROM content_sources cs
      JOIN documents d ON d.program_id = cs.program_id
      WHERE d.id = dv.document_id
        AND cs.name = 'Legacy migration source'
      LIMIT 1
    )
  ),
  source_origin_uri = COALESCE(
    dv.source_origin_uri,
    'legacy-object-storage://' || COALESCE(dv.source_url, dv.id::text)
  ),
  source_owner = COALESCE(dv.source_owner, 'Legacy owner review required'),
  original_file_name = COALESCE(
    dv.original_file_name,
    nullif(regexp_replace(COALESCE(dv.source_url, ''), '^.*/', ''), ''),
    'legacy-document'
  ),
  lifecycle_state = CASE WHEN dv.is_active THEN 'active' ELSE 'retired' END,
  scan_status = CASE
    WHEN dv.scan_status IN ('clean', 'infected', 'unavailable', 'error') THEN dv.scan_status
    ELSE 'legacy_accepted'
  END,
  scan_completed_at = COALESCE(dv.scan_completed_at, dv.uploaded_at, now()),
  approved_at = CASE
    WHEN dv.is_active THEN COALESCE(dv.approved_at, dv.uploaded_at, now())
    ELSE dv.approved_at
  END,
  approval_notes = CASE
    WHEN dv.is_active THEN COALESCE(
      dv.approval_notes,
      'Legacy active content backfilled during P0/P1 rollout; provenance and scan review required'
    )
    ELSE dv.approval_notes
  END,
  activated_at = CASE
    WHEN dv.is_active THEN COALESCE(dv.activated_at, dv.uploaded_at, now())
    ELSE dv.activated_at
  END,
  retired_at = CASE
    WHEN NOT dv.is_active THEN COALESCE(dv.retired_at, now())
    ELSE dv.retired_at
  END
WHERE
  dv.source_id IS NULL
  OR dv.source_origin_uri IS NULL
  OR dv.source_owner IS NULL
  OR dv.original_file_name IS NULL;

ALTER TABLE document_versions
  ALTER COLUMN source_id SET NOT NULL,
  ALTER COLUMN source_origin_uri SET NOT NULL,
  ALTER COLUMN source_owner SET NOT NULL,
  ALTER COLUMN original_file_name SET NOT NULL;

DO $$
BEGIN

  ) THEN

      lifecycle_state IN (

        'retired', 'quarantined', 'rejected', 'revoked', 'failed'
      )
    );
  END IF;
  IF NOT EXISTS (
    SELECT 1 FROM pg_constraint WHERE conname = 'document_versions_classification_check'
  ) THEN
    ALTER TABLE document_versions ADD CONSTRAINT document_versions_classification_check CHECK (
      classification IN ('public', 'internal', 'confidential', 'restricted')
    );
  END IF;
  IF NOT EXISTS (
    SELECT 1 FROM pg_constraint WHERE conname = 'document_versions_scan_status_check'
  ) THEN
    ALTER TABLE document_versions ADD CONSTRAINT document_versions_scan_status_check CHECK (
      scan_status IN (
        'pending', 'running', 'clean', 'infected', 'unavailable',
        'error', 'legacy_accepted'
      )
    );
  END IF;
  IF NOT EXISTS (
    SELECT 1 FROM pg_constraint WHERE conname = 'document_versions_active_control_check'
  ) THEN
    ALTER TABLE document_versions ADD CONSTRAINT document_versions_active_control_check CHECK (
      is_active = false OR (
        lifecycle_state = 'active'
        AND approved_at IS NOT NULL
        AND source_id IS NOT NULL

      )
    );
  END IF;
  IF NOT EXISTS (
    SELECT 1 FROM pg_constraint WHERE conname = 'document_versions_separation_check'
  ) THEN
    ALTER TABLE document_versions ADD CONSTRAINT document_versions_separation_check CHECK (
      approved_by IS NULL OR uploaded_by IS NULL OR approved_by::text <> uploaded_by
    );
  END IF;
  IF NOT EXISTS (
    SELECT 1 FROM pg_constraint WHERE conname = 'documents_lifecycle_check'

END $$;

  p_resource_id text,
  p_request_id text,
  p_source_ip text,
  p_details jsonb
)
RETURNS TABLE(id uuid, occurred_at timestamptz, event_hash text)
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public, pg_catalog
AS $$
DECLARE
  v_id uuid := gen_random_uuid();
  v_occurred_at timestamptz := clock_timestamp();
  v_previous_hash text;
  v_event_hash text;
BEGIN
  IF p_action IS NULL OR p_action = '' THEN
    RAISE EXCEPTION 'security event action is required';
  END IF;
  IF p_outcome NOT IN ('success', 'denied', 'failure') THEN
    RAISE EXCEPTION 'security event outcome is invalid';
  END IF;

  PERFORM pg_advisory_xact_lock(hashtext('truenote.security_events.hash_chain'));
  SELECT se.event_hash
  INTO v_previous_hash
  FROM security_events se
  ORDER BY se.sequence DESC
  LIMIT 1;

  v_event_hash := encode(
    digest(
      concat_ws(
        '|',
        COALESCE(v_previous_hash, ''),
        v_id::text,
        v_occurred_at::text,
        p_action,
        p_outcome,
        COALESCE(p_actor_user_id::text, ''),
        COALESCE(p_actor_email, ''),
        COALESCE(p_actor_role, ''),
        COALESCE(p_program_id::text, ''),
        COALESCE(p_resource_type, ''),
        COALESCE(p_resource_id, ''),
        COALESCE(p_request_id, ''),
        COALESCE(p_source_ip, ''),
        COALESCE(p_details, '{}'::jsonb)::text
      ),
      'sha256'
    ),
    'hex'
  );

  INSERT INTO security_events (
    id, occurred_at, action, outcome, actor_user_id, actor_email, actor_role,
    program_id, resource_type, resource_id, request_id, source_ip, details,
    previous_hash, event_hash
  ) VALUES (
    v_id, v_occurred_at, p_action, p_outcome, p_actor_user_id, p_actor_email,
    p_actor_role, p_program_id, p_resource_type, p_resource_id, p_request_id,
    p_source_ip, COALESCE(p_details, '{}'::jsonb), v_previous_hash, v_event_hash
  );

  RETURN QUERY SELECT v_id, v_occurred_at, v_event_hash;
END;
$$;

-- Transactional audit for every document-version lifecycle transition.
CREATE OR REPLACE FUNCTION audit_document_version_lifecycle()
RETURNS trigger
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public, pg_catalog
AS $$
DECLARE
  v_program_id uuid;
  v_actor_id uuid;
  v_actor_email text;
  v_actor_role text;
  v_actor_text text;
  v_old_state text;
BEGIN
  SELECT d.program_id INTO v_program_id
  FROM documents d
  WHERE d.id = NEW.document_id;

  v_old_state := CASE WHEN TG_OP = 'INSERT' THEN NULL ELSE OLD.lifecycle_state END;
  v_actor_text := CASE NEW.lifecycle_state
    WHEN 'active' THEN NEW.approved_by::text
    WHEN 'rejected' THEN NEW.rejected_by::text
    WHEN 'revoked' THEN NEW.revoked_by::text
    ELSE NEW.uploaded_by
  END;
  IF v_actor_text ~* '^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$' THEN
    v_actor_id := v_actor_text::uuid;
    SELECT u.email, u.role::text INTO v_actor_email, v_actor_role
    FROM users u WHERE u.id = v_actor_id;
  END IF;

  PERFORM append_security_event(
    'document.lifecycle.' || NEW.lifecycle_state,
    'success',
    v_actor_id,
    v_actor_email,
    v_actor_role,
    v_program_id,
    'document_version',
    NEW.id::text,
    NULL,
    NULL,
    jsonb_build_object(
      'documentId', NEW.document_id,
      'versionNumber', NEW.version_number,
      'previousState', v_old_state,
      'newState', NEW.lifecycle_state,
      'classification', NEW.classification,
      'scanStatus', NEW.scan_status,
      'isActive', NEW.is_active
    )
  );
  RETURN NEW;
END;
$$;

DROP TRIGGER IF EXISTS document_versions_audit_insert ON document_versions;
CREATE TRIGGER document_versions_audit_insert
AFTER INSERT ON document_versions
FOR EACH ROW EXECUTE FUNCTION audit_document_version_lifecycle();

DROP TRIGGER IF EXISTS document_versions_audit_lifecycle ON document_versions;
CREATE TRIGGER document_versions_audit_lifecycle
AFTER UPDATE OF lifecycle_state ON document_versions
FOR EACH ROW
WHEN (OLD.lifecycle_state IS DISTINCT FROM NEW.lifecycle_state)
EXECUTE FUNCTION audit_document_version_lifecycle();

-- Transactional audit for source-registry creation/retirement.
CREATE OR REPLACE FUNCTION audit_content_source_change()
RETURNS trigger
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public, pg_catalog
AS $$
DECLARE
  v_actor_id uuid;
  v_actor_email text;
  v_actor_role text;
BEGIN
  v_actor_id := COALESCE(NEW.approved_by, NEW.created_by);
  SELECT u.email, u.role::text INTO v_actor_email, v_actor_role
  FROM users u WHERE u.id = v_actor_id;
  PERFORM append_security_event(
    CASE WHEN TG_OP = 'INSERT' THEN 'content_source.created' ELSE 'content_source.changed' END,
    'success',
    v_actor_id,
    v_actor_email,
    v_actor_role,
    NEW.program_id,
    'content_source',
    NEW.id::text,
    NULL,
    NULL,
    jsonb_build_object(
      'name', NEW.name,
      'originType', NEW.origin_type,
      'ownerName', NEW.owner_name,
      'isActive', NEW.is_active,
      'approvedAt', NEW.approved_at
    )
  );
  RETURN NEW;
END;
$$;

CREATE TRIGGER content_sources_audit_insert
AFTER INSERT ON content_sources
FOR EACH ROW EXECUTE FUNCTION audit_content_source_change();
