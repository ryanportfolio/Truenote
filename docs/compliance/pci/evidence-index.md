## Evidence quality and handling

- Link every artifact to scope, owner, date, environment, commit/release, result,
  finding disposition, and next review/expiry.
- Retain raw machine-readable results when safe; retain executive summaries for
  broad distribution and detailed exploit material only in restricted storage.
- Do not treat a screenshot as runtime acceptance when a synthetic transaction or
  configuration API export is available.
- Never store PAN, credentials, customer content, raw prompts, or production dumps
  in repository evidence.

## Acceptance test

For a sampled control, an assessor can follow the index from requirement to owner,
approved procedure, current configuration, execution receipt, finding disposition,
and retest without relying on verbal explanation.
